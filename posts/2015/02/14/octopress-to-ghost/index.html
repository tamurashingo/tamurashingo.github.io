OctopressからGhostへ移行

tamuraです。 blogシステムをOctopressからGhostに移行しました。

Octopress

手元ですべてのHTMLを生成するのでGithubとかでブログを始めるのにちょうどいいやつです。手元でEmacsでMarkdownで書いて、プレビューで確認して、よしOKとなったらdeployです。技術者にはそこそ>こ良いブログシステムだと思います。

突然のQiita

Qiitaはすごく便利です。左側に生Markdownを書くと右側でプレビューが確認できます。 Octpressだとコンパイル待ちがある言語（たとえばCとかJava）で開発しているような感じですが、QiitaはREPLがある言語（LispやRubyなどなど）で開発しているような感じです。何かまずい書き方をして>いたらすぐに確認できる。 Qiitaに何回か投稿していると、Octopressが非常に面倒なものに見えてきます。また、何人かで技術系ブログを書いていきたいと思っていたいのですが、Octopressの環境をローカルで準備してもらうのがち ょっと手間で、気軽に「書いて」と頼みにくいです。

そしてブログの乗り換えの検討

幸いにしてサーバを用意できているので、別にすべてがHTMLである必要はありません。いくつかブログシステムを比較検討してみました。

ポイントは

  • 手元に言語を入れなくても良い
  • ブラウザから投稿できる
  • マルチユーザに対応している

という点です。

Ghost

いろいろ調べるとこんなものを見つけました。アクセスのたびにHTMLを生成するようなのですが、なかなか良いと思いました。

  • Good
    • Markdownで書ける
    • Qiitaのようにリアルタイムでプレビューが出る
    • マルチユーザ対応
    • JSで動いている
    • 最悪何かあったら対処できる可能性がある
  • Bad
    • シンタックスハイライトがない
    • 別な方法で対応可能
    • 未知数

WordPress

ブログといえばWordPressです。なんか導入が大変そうと思って早々に却下しました。

  • PHPで動いている
    • PHPをディスるわけではなく、PHPについてあまり知らないので何かあったら対処できないためです
  • 重そう
    • 借りているサーバ環境だとギリギリ動くといった感じです

あとは

オープンソース・フリーなCMS一覧、比較

というまとめを見つけましたが、Ghostを見つけてしまっていたのでこれでいいやとなりました。

実際の乗り換え

記事

Markdown形式なのでひたすらコピペでOKです。というか元々記事が全然なかったのでそれほど大変ではありませんでした。

シンタックスハイライト

Octopressはバッククオート3つ + 言語名でシンタックスハイライトを行ってくれました。

```java
public class Helloworld {
    public static void main(String...args) {
        System.out.println("hello world");
    }
}

Ghostはそういう機能は今のところありません。が、google-code-prettifyを使うとやってくれます。 具体的には https://cyanbyfuchsia.wordpress.com/2014/01/02/add-code-syntax-highlighting-to-your-ghost-blogging-site/ にあるようにするだけです。

```prettyprint lang-java
public class Helloworld {
    public static void main(String...args) {
        System.out.println(\"hello world\");
    }
}

こう書くと

public class Helloworld {
    public static void main(String...args) {
        System.out.println(\"hello world\");
    }
}

こうなります。
